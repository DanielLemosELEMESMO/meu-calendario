## Analise de Arquitetura e SOLID
- **SRP (Single Responsibility Principle):** Componentes de UI apenas renderizam e disparam acoes; a logica de dados fica isolada em servicos/repositories.
- **OCP (Open/Closed Principle):** O contrato `EventRepository` permite trocar a implementacao mock por uma real sem alterar a UI.
- **DIP (Dependency Inversion Principle):** A UI depende da abstracao (interface) e recebe a implementacao via injecao simples no app.
- **Separacao de Camadas:** `components/` para UI, `features/` para telas, `data/` para repositorios e cache, `models/` para tipos.

## Observacoes
- A estrategia de cache inicial usara `localStorage` por simplicidade. Se precisarmos de consultas mais complexas, podemos migrar para IndexedDB sem mudar o contrato do repositorio.
- Decisao: manter `localStorage` nesta versao para reduzir complexidade; a troca para backend sera feita via `EventRepository`, sem impactar a UI.
